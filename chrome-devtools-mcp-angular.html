01 · El Problema

Los agentes de IA programan
con los ojos vendados.

🔄
Loop manual interminable El IA genera código → tú abres el navegador → copias el error → lo pegas de vuelta → repites.
🎯
Sin contexto de ejecución real El modelo no sabe si el componente renderizó, si la petición falló con 403 o si el LCP está en 8 segundos.
🧠
Alucinaciones de estado El agente asume cómo se ve tu app en vez de observarla, lo que genera fixes incorrectos una y otra vez.
⏱️
Tiempo perdido en contexto Cada ciclo de feedback requiere que tú seas el puente entre el browser y el agente.
02 · La Solución

Chrome DevTools
MCP Server

Un servidor MCP oficial del equipo de Chrome que expone las capacidades de DevTools a cualquier asistente de IA compatible. Lanzado en public preview el 23 de septiembre de 2025, con auto-connection desde la v0.12.1.

🔌

Model Context Protocol

Estándar open-source de Anthropic para conectar LLMs a herramientas externas. El "USB-C" de las integraciones de IA.

🧩

Servidor MCP oficial

Mantenido por el equipo de Chrome DevTools. Disponible vía npx, sin instalación permanente.

🔁

Loop cerrado

El agente observa, actúa y verifica en el browser real. Sin intermediarios humanos en el ciclo de debug.

03 · Integración

Setup en 3 pasos

1

Requisitos

Node.js v20.19+ · Chrome estable · Cliente MCP (Claude Desktop, Cursor, VS Code Copilot)

2

Configurar el cliente MCP

Agrega esta entrada en tu archivo de config (claude_desktop_config.json o .cursor/mcp.json):

{
  "mcpServers": {
    "chrome-devtools": {
      "command": "npx",
      "args": ["chrome-devtools-mcp@latest"]
    }
  }
}
3

Verificar que funciona

Con tu app Angular corriendo en localhost:4200, lanza este prompt en tu agente:

→ "Navega a localhost:4200 y toma un screenshot."
